# Task List App (Agile Group Project)

## What this is

This was a group project for my Agile Software Development class. My team built a task list application in C++ over the course of a 16-week semester using a sprint-based workflow.

This project was less about creating a huge polished final product and more about learning how to work through iterative development as a team.

## Why I built it

This project was part of an Agile course, so the focus was on collaboration, sprint planning, and building features over time instead of trying to do everything at once. There were three of us on the team, and we completed 6 active sprints during the semester.

## What it does

* Creates and manages a basic task list
* Demonstrates team-based C++ development
* Includes test files created as part of the project
* Reflects an Agile workflow across multiple sprints

## Tech used

* C++
* Catch2 (`catch.hpp`) for testing
* Azure DevOps
* Agile / Scrum-style sprint planning

## Project files

This repository currently includes:

* `main.cpp`
* `test.cpp`
* `catch.hpp`

## Project status

This project is being included as a coursework portfolio piece and development snapshot. At this point, the codebase is not fully polished, and parts of the main program are currently commented out from earlier development/testing. I’m keeping it here to reflect the actual class project process and the work that went into it.

## My role

I served as the Scrum Master for our team of three while also contributing to development. This meant I helped organize sprint planning, kept the team on track throughout each iteration, and made sure we were progressing across all 6 sprints.

Alongside that, I contributed to coding, testing, and debugging features as the project evolved.

## What I learned

This project gave me hands-on experience with Agile development in an actual team setting. I learned how to:

* Work through sprint-based development over time
* Collaborate with other developers on shared code
* Use Azure DevOps as part of the project workflow
* Write and work with tests in C++
* Adjust when things didn’t go exactly as planned

## Challenges

One of the biggest challenges was balancing the coding side with the Agile process itself. It wasn’t just about making the app work — it was also about communicating, planning, and keeping progress moving across multiple sprints. Like a lot of class projects, not everything ended in a perfectly clean final state, but it was still a valuable learning experience.

I’d like to:
